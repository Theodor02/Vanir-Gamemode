ix.playerEffects.SetExclusive(identifier, identifier, string)
plugins/player_effects/sh_api.lua:45

Declare two effect identifiers as mutually exclusive. When a player already has a modifier with identifierA and a new modifier with identifierB is added (or vice-versa), the conflict is resolved according to the chosen resolution strategy. "priority" – higher opts.priority wins; ties keep the existing effect "first" – the already-applied effect always stays "last" – the newly-applied effect always wins
